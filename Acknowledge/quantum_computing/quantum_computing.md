## 量子比特

### 与经典比特的对比

在我们的电脑、手机（经典计算机）里，信息的最小单位是**比特（Bit）**。它是**确定**的，其状态只能是 0 和 1。

就像硬币平放在桌面上，要么是“正面”，要么是“反面”。

而量子比特（Qubit）就像一枚**"旋转的硬币"**。

在量子计算机里，信息的最小单位是**量子比特（Quantum Bit，简称 Qubit）**。它的状态不是确定的，而是有概率的。它可以是 0 ，也可以是 1，更重要的是，它可以**同时处于 0 和 1 的叠加状态**。

他就像一枚快速旋转的硬币，既包含了正面的成分，也包含了反面的成分。这就是量子比特的**叠加态**。

### 特性

#### **叠加态 (Superposition)**

经典比特一次只能走一条路，但是量子比特可以“分身”，同时走所有的死胡同和正确的路。这使得量子计算机在处理某些问题（如搜索）时，能并行处理大量可能性。

我们通常用 $|0\rangle$ 表示 0 态，用 $|1\rangle$ 表示 1 态。叠加态写成：$a|0\rangle + b|1\rangle$（意为：有 $a$ 的概率幅是0，有 $b$ 的概率幅是1）。

#### **测量坍缩 (Collapse)**

以硬币为例，当硬币还在转（叠加态）时，它同时是0和1。你用手拍住它（测量），硬币瞬间停止旋转，变成确定的“正面”或“反面”。

**结论**：观测会改变系统状态。一旦你“看”了量子比特，它就从“神奇的叠加态”变回了“普通的经典比特”。

#### **纠缠态 (Entanglement)**

这是量子力学最诡异的现象，爱因斯坦称之为“鬼魅般的超距作用”。

想象有两枚纠缠的硬币，一枚在地球，一枚在火星。只要它们处于纠缠态，无论距离多远，只要你拍下地球的那枚硬币发现是“正面”，火星的那枚硬币**瞬间**也会变成“正面”（或者反面，取决于设置）。它们之间仿佛有心灵感应。

| 特性         | 经典比特 (Bit)     | 量子比特 (Qubit)       |
| ------------ | ------------------ | ---------------------- |
| **基础状态** | 0 或 1             | $                      |
| **实际状态** | 必须确定（非0即1） | 可以叠加（同时是0和1） |
| **操作类比** | 翻转开关           | 旋转球体               |
| **算力增长** | 线性增长 (1+1=2)   | 指数级爆炸 (2的N次方)  |

## 叠加态

### 直观解释

叠加态是量子比特的一个十分特殊的状态。

假设你要走出一个极其复杂的迷宫。

- **经典比特**：老鼠到一个岔路口，只能选择左或右。如果走错了，它必须退回来重走。它一次只能试一条路。
- **量子比特**：想象把水倒进迷宫。水流到岔路口时，不是选左或右，而是**分流**，同时流向左边和右边。
  - **叠加态**就是这种“同时处于多条路径”的状态。
  - 这意味着量子计算机可以**同时探索所有可能的答案**，而不是像经典计算机那样一个接一个地试。

### 本质

量子力学告诉我们，微观粒子（电子、光子）具有**波粒二象性**。它们不仅是粒子，也是波。

**而量子比特的叠加态，本质上就是概率波的叠加。**

这也是为什么量子算法（如Grover搜索）能生效：它通过精妙的算法设计，让错误答案的概率波互相抵消（相消干涉），让正确答案的概率波互相叠加增强（相长干涉）。

### 数学表示

在 PyQPanda 或任何量子编程中，我们必须用数学来描述它。经典比特的状态只有 $0$ 或 $1$。

量子比特的状态 $|\psi\rangle$ 写成：
$$
|\psi\rangle = \alpha |0\rangle + \beta |1\rangle
$$
这里的 $|0\rangle$ 和 $|1\rangle$ 代表确定的基态（就像地图上的北和南）。而 $\alpha$ (阿尔法) 和 $\beta$ (贝塔) 是两个复数，称为**概率幅**。

**注意：**

1. **不是直接的概率**：$\alpha$ 和 $\beta$ 本身不是概率，它们可以有正负，甚至是虚数（这允许了波的抵消）

2. **平方才是概率**：

   *   测量得到 0 的概率 = $|\alpha|^2$
   *   测量得到 1 的概率 = $|\beta|^2$

3. **归一化**：概率总和必须是 1
   $$
   |\alpha|^2 + |\beta|^2 = 1
   $$

以 H 门为例：

PyQPanda 中最常用的叠加态是 H 门产生的状态，
$$
|\psi\rangle = \frac{1}{\sqrt{2}}|0\rangle + \frac{1}{\sqrt{2}}|1\rangle
$$

*    $\alpha = \frac{1}{\sqrt{2}}$，$\beta = \frac{1}{\sqrt{2}}$
*   测量得到 0 的概率 = $(\frac{1}{\sqrt{2}})^2 = \frac{1}{2} = 50\%$
*   测量得到 1 的概率 = $50\%$

### 作用

 叠加态的最大魅力在于**指数级并行**。

*   1个量子比特：可以同时处于 2 个状态（0, 1）的叠加
*   2个量子比特：可以同时处于 4 个状态（00, 01, 10, 11）的叠加
*   3个量子比特：可以同时处于 8 个状态的叠加
*   ...
*   50个量子比特：可以同时处于 $2^{50}$ 个状态的叠加。这个数字大约是 **1125万亿**！

经典计算机处理这1125万亿个数据，需要循环1125万亿次。量子计算机**只需要操作这50个比特1次**，就相当于同时操作了这1125万亿个数据。这就是所谓的**量子并行计算（Quantum Parallelism）**。

### 示例

以 PyQPanda 框架编写代码：

```

```

```python
from pyqpanda import *

def superposition_practice():
    machine = init_quantum_machine(QMachineType.CPU)
    q = machine.qAlloc_many(1)
    c = machine.cAlloc_many(1)
    prog = QProg()
    
    # 第一次 H 门： |0>  --->  0.7*|0> + 0.7*|1> (叠加态)
    prog.insert(H(q[0]))
    
    # 第二次 H 门： 叠加态 --->  ? 
    prog.insert(H(q[0]))
    
    # 测量
    prog.insert(measure_all(q, c))
    
    result = machine.run_with_configuration(prog, c, 2000)
    print("两次H门后的结果:", result)
    machine.finalize()

if __name__ == "__main__":
    superposition_practice()
# 两次H门后的结果: {'0': 2000}
```

实验解析：
*   如果是抛硬币，你抛一次是随机，抛两次还是随机
*   但在量子计算中，你会发现结果几乎 **100% 是 '00'**（即回到原点）
*   **原因**：由于**波的干涉**，第二次 H 门操作让 $|1\rangle$ 的概率波发生了相消干涉（抵消了），而 $|0\rangle$ 的概率波发生了相长干涉（增强了）。

## 量子纠缠

如果说“叠加态”是一个量子比特的个人秀，那么“纠缠态”就是两个或多个量子比特之间的心灵感应。

### 定义

假设你有两枚硬币（量子比特 A 和 B）。

**在经典世界里**，如果你把它们分开到银河系的两端，你翻开 A 发现是正面，这完全不会影响 B 的结果。B 是正面还是反面，早在你分开它们时或者随机投掷时就决定了，两者互不相干。

但是在**量子世界**，如果 A 和 B 处于**纠缠态**，它们就变成了一个不可分割的整体系统。无论它们相距多远（一个在地球，一个在火星），当你观测 A 并发现它是正面的瞬间，B 会**立刻、瞬间**坍缩成正面（或者反面，取决于纠缠的方式）。

这种关联不是因为它们事先商量好了，而是它们在观测前都没有确定状态，一旦观测其中一个，另一个的状态也被同时锁定了。

爱因斯坦曾极力反对这个概念，称之为 “Spooky action at a distance”。但后来的实验（如阿斯佩实验、2022年诺贝尔物理学奖相关实验）证明了量子纠缠是真实存在的。

### 数学表示

#### **1. 系统的状态表示**
如果是两个量子比特，它们的状态可以用向量表示。
*   $|00\rangle$：两个都是0
*   $|01\rangle$：A是0，B是1
*   $|10\rangle$：A是1，B是0
*   $|11\rangle$：两个都是1

#### **2. 叠加态**
我们来看两种状态：

**状态 A（非纠缠态/直积态）：**
$$
 |\psi\rangle = \frac{1}{2} (|00\rangle + |01\rangle + |10\rangle + |11\rangle) 
$$
乍一看很复杂，但其实它可以拆解成：
$$
 (\frac{|0\rangle + |1\rangle}{\sqrt{2}}) \otimes (\frac{|0\rangle + |1\rangle}{\sqrt{2}}) 
$$
这意味着，第一个比特处于叠加态，第二个比特也处于叠加态。它们是独立的，你测量第一个比特，完全不影响第二个比特的概率。

**状态 B（纠缠态/贝尔态）：**
$$
 |\Phi^+\rangle = \frac{|00\rangle + |11\rangle}{\sqrt{2}} 
$$
这个状态的意思是，系统有 50% 概率是 $|00\rangle$，有 50% 概率是 $|11\rangle$。**且绝对不存在 $|01\rangle$ 或 $|10\rangle$ 的情况。**

**数学判定：**如果你试图把状态 B 拆解成 $(a|0\rangle + b|1\rangle) \otimes (c|0\rangle + d|1\rangle)$，你会发现**无解**。无法写成两个独立比特的乘积，这就是**纠缠**的数学定义。

### 重要性

1. **量子隐形传态**：利用纠缠，可以将一个量子态从一处传输到另一处，而不需要传输粒子本身。
2. **超级算力**：在Shor算法或Grover算法中，我们操纵的是所有比特纠缠在一起的巨大向量空间。没有纠缠，量子计算机就只是一堆并行运行的普通计算器，无法产生指数级的加速。
3. **量子加密**：如果有人试图窃听处于纠缠态的光子，纠缠状态就会被破坏（被测量），通信双方立刻就能发现有人窃听（如 BB84 协议的变种 E91 协议）。

###  PyQPanda 实践：制造贝尔态

在编程中，制造纠缠态的标准步骤是：**H门 + CNOT门**

*   **H门**：让第一个比特进入叠加态（制造随机性）
*   **CNOT门（受控非门）**：把两个比特绑在一起。如果控制位是0，目标位不变；如果控制位是1，目标位翻转。

计算：
1.  初始：$|00\rangle$
2.  对第一个比特做 H：$(\frac{|0\rangle+|1\rangle}{\sqrt{2}})|0\rangle = \frac{|00\rangle + |10\rangle}{\sqrt{2}}$
3.  以第一个为控制，第二个为目标做 CNOT：
    *   $|00\rangle$ 中第一位是0 -> 第二位不变 -> $|00\rangle$
    *   $|10\rangle$ 中第一位是1 -> 第二位翻转 -> $|11\rangle$
4.  最终结果：$\frac{|00\rangle + |11\rangle}{\sqrt{2}}$

```python
from pyqpanda import *

def create_bell_state():
    machine = init_quantum_machine(QMachineType.CPU)
    q = machine.qAlloc_many(2)
    c = machine.cAlloc_many(2)
    prog = QProg()

    # 构建贝尔态电路
    # 第一步：H门作用于 q[0]，创造叠加
    prog.insert(H(q[0]))
    
    # 第二步：CNOT门，q[0]控制q[1]
    # 在PyQPanda中，CNOT(control, target)
    prog.insert(CNOT(q[0], q[1]))

    # 测量
    prog.insert(measure_all(q, c))

    # 理论预期：只能测到 00 或者 11，各占50%。
    # 不应该出现 01 或 10 (理想情况下)。
    result = machine.run_with_configuration(prog, c, 1000)
    
    print("贝尔态测量结果 (预期只有00和11):", result)
    
    machine.finalize()

if __name__ == "__main__":
    create_bell_state()
```

测量结果：

```bash
贝尔态测量结果 (预期只有00和11): {'00': 470, '11': 530}
```

这证明了 q[0] 和 q[1] 不再是独立的。一旦 q[0] 变成了 1，q[1] 就被迫变成了 1。

## GHZ 态

GHZ态（Greenberger–Horne–Zeilinger State）是展示多粒子的量子纠缠特性的**经典案例**。

### 定义

单个量子比特叠加是“既是0又是1”，而 **GHZ态** 则是让三个硬币进入一种命运共同体的状态。

状态的数学表示：
$$
\frac{|000\rangle + |111\rangle}{\sqrt{2}}
$$
即这三个量子比特，要么全都是 0，要么全都是 1。

而神奇之处是，一旦你测量了其中**任何一个**比特（比如发现它是0），另外两个比特**瞬间**也会坍缩成0，无论它们相隔多远。绝不会出现 “010” 或 “100” 这种存在某个量子比特的状态和其他比特不同的情况。

### 电路构建

要制造这种状态，我们需要用到两个核心逻辑门：
1.  **H 门（Hadamard）**：制造叠加，让第一个比特转起来
2.  **CNOT 门（受控非门）**：制造关联，如果控制位是1，就把目标位翻转。

**实现步骤**：

1.  初始状态：$|000\rangle$（三个比特都是0）
2.  第一步：对 q0 施加 H门
    *   q0 变成叠加态 $(|0\rangle + |1\rangle)$
    *   系统状态变成：$\frac{|000\rangle + |100\rangle}{\sqrt{2}}$（注意这里q1, q2还没动）
3.  第二步：对 q0（控）和 q1（受）施加 CNOT 门
    *   当 q0 是 0 时，q1 不动（0）；当 q0 是 1 时，q1 翻转（0->1）
    *   系统状态变成：$\frac{|000\rangle + |110\rangle}{\sqrt{2}}$（此时 q0 和 q1 发生纠缠）
4.  第三步：对 q1（控）和 q2（受）施加 CNOT 门
    *   当 q1 是 1 时，q2 也翻转成 1
    *   系统状态最终变成：$\frac{|000\rangle + |111\rangle}{\sqrt{2}}$（GHZ态）

### PyQPanda 实现

```python
from pyqpanda import *


def ghz_state_practice():
    machine = init_quantum_machine(QMachineType.CPU)
    q = machine.qAlloc_many(3)
    c = machine.cAlloc_many(3)

    prog = QProg()

    # 第一步：让 q0 进入叠加态
    prog.insert(H(q[0]))

    # 第二步：将 q0 的状态传染给 q1
    prog.insert(CNOT(q[0], q[1]))

    # 第三步：将 q1 的状态传染给 q2
    prog.insert(CNOT(q[1], q[2]))

    prog.insert(measure_all(q, c))

    print("正在运行 GHZ 态生成电路...")
    result = machine.run_with_configuration(prog, c, 1000)

    print("运行结果分布: ", result)

    machine.finalize()


if __name__ == "__main__":
    ghz_state_practice()
```

## Shor 算法

Shor 算法是量子计算皇冠上的明珠，它最著名的应用就是**分解大整数**。

### 原理

Shor 算法利用量子计算机的并行计算能力算出所有可能的结果，再利用 QFT 制造波的干涉，把错误的答案抵消掉，只留下那个唯一的正确答案（周期 r）。

### 大整数分解

经典计算机分解整数（比如把 15 分解成 3x5）很慢，因为它是靠试除法或者复杂的筛选法。

而Shor 算法天才的地方在于，它利用数论将 “分解整数 $N$” 的问题，转化为了一个 “寻找周期（求阶）” 的问题。

#### **求解步骤**

1. 本地 PC 随机选一个小于 $N$ 的数 $a$ 。此处以 $N=15$， $a=7$为例。

2. 量子计算机寻找函数 $f(x) = a^x \pmod N$ 的周期 $r$。

   *   比如 $7^1 \mod 15 = 7$

   *   $7^2 \mod 15 = 4$

   *   $7^3 \mod 15 = 13$

   *   $7^4 \mod 15 = 1$

   *   $7^5 \mod 15 = 7$ ...（开始循环）

   *   这里周期 $r=4$。量子计算机擅长瞬间找到这个 $r$

3. 本地 PC 利用 $r$ 计算出 $N$ 的因子：$gcd(a^{r/2} \pm 1, N)$ 就能得到因子

对于经典电脑实现大数分级，找周期需要计算无数次乘方，非常慢。而量子计算机利用**叠加态**，一次性输入所有的 $x$，并行计算所有的 $a^x$，然后利用量子傅里叶变换（QFT）把周期 $r$ 提取出来。其计算速度是经典电脑计算的**指数级效率**。

#### **寻找周期的意义**

Shor 算法构建了两个寄存器（两排量子比特）：
*   寄存器 A（输入）：存 $x$
*   寄存器 B（输出）：存 $f(x) = a^x \pmod N$

**1. 制造全知的叠加态**

我们不对 $x$ 一个一个试。我们对寄存器 A 使用 **H 门**，让它同时处于 $0$ 到 $2^n-1$ 的所有状态。
然后运行 $f(x)$ 模块。此时，系统状态变成了：
$$
 \sum |x\rangle \otimes |a^x \pmod N\rangle 
$$

这意味着量子计算机在这一瞬间，**同时**算出了所有的 $x$ 和对应的余数

**2. 隐形的周期结构**

虽然我们算出了所有结果，但它们是纠缠在一起的。

那么假设我们此时测量一下寄存器 B，得到了一个余数，比如 $1$。则寄存器 A 会发生**坍缩**，它只剩下那些能算出余数 1 的 $x$ 值：$|0\rangle, |4\rangle, |8\rangle, |12\rangle \dots$

你看，这些留下的数字之间，间隔正好是周期 $r=4$

但是我们不能直接读寄存器 A。因为如果直接测量，你只会随机得到 $0$ 或 $4$ 或 $8$ 中的**某一个**数字。光凭一个数字，你不知道它们之间的间隔（周期）是多少。

**此时就需要使用 QFT 将状态循环的周期给提取出来**。

### QFT 

量子傅里叶变换（QFT） 的作用是将状态从**时域**（具体的数字序列）转换到**频域**（数字出现的频率）。

**实现方法**：

**1. 波的干涉**

假设寄存器 A 里的状态 $|0\rangle + |4\rangle + |8\rangle + \dots$ 是一组波源。如果我们不做处理直接测量，就像在混乱的水面上随机捞点，看不出规律。

而QFT 就像一个透镜。它会对这些波应用相位旋转。

*   对于错误的周期猜测（比如 3, 5），波峰和波谷会相互抵消（相消干涉），**概率变为 0**。
*   对于正确的周期倒数（$1/r$ 的倍数），所有的波峰会叠加在一起（相长干涉），**概率被极度放大**。

**2. 测量结果**

应用 QFT 后，我们测量寄存器 A。我们得到的不再是 $x$ 本身，而是与周期 $r$ 相关的数值 $y$。

数学关系是：
$$
\frac{y}{Q} \approx \frac{k}{r}
$$
其中：

*   $y$ 是测量结果
*   $Q$ 是寄存器能存的最大数，比如 $2^{10}$
*   $k$ 是某个整数
*   $r$ 就是我们要找的周期

**3. 反推**

拿到 $y$ 后，我们在本地 PC 上做一个简单的除法和连分数展开，就能算出分母 $r$ 是多少。

### 总结

Shor 算法的计算流程：

1.  并行计算：利用叠加态，一次性算出所有 $a^x$
2.  周期蕴含：此时，正确的周期性已经隐藏在量子态的振幅分布中了，但直接看是看不见的（
3.  QFT 聚焦：利用 QFT 进行基变换，引发**量子干涉**。就像用棱镜把混合光分开一样，它把分散在各个 $x$ 上的概率，全部汇聚到了 $k/r$ 也就是周期相关的这几个状态上
4.  测量它，就能以极高的概率直接得到周期的线索。

Shor 算法能比经典算法快指数倍的核心原因：它不是在瞎猜，它是利用波的干涉原理，让错误的答案自己消失，让正确的答案自己浮现。

### 经典分解算法的复杂度

在经典计算机上，分解大整数是非常困难的

最简单的算法是试除法：从 2 开始一直除到 $\sqrt{N}$。

如果 $N$ 是一个 $n$ 位的二进制数（即 $N \approx 2^n$），那么需要尝试大约 $2^{n/2}$ 次。

*   复杂度：**指数级** $O(2^{n/2})$，或者记作 $O(\sqrt{N})$
*   直观理解：$N$ 每增加 1 个比特（比如从 100 位变成 101 位），计算量就会翻倍甚至更多。

目前最强的经典算法是通用数域筛法 GNFS：它比试除法快很多，是亚指数级的，但依然非常慢。
*   复杂度：大约是 $\exp(n^{1/3})$ 级别
*   现状：分解一个 2048 位的 RSA 密钥，用目前最强的超级计算机可能需要数百万年甚至更久

### Shor 实现指数级运行的原因

Shor 算法将复杂度降低到了多项式级别 $O(n^3)$，$n$ 是比特数。其核心原因在于量子并行性与干涉的结合。

- **量子并行性**：利用量子比特的叠加态，使用一个寄存器存储所有可能的取值 $x$ ($|0\rangle + |1\rangle + \dots + |2^n-1\rangle$)。然后使用**一次**模幂运算，就能够**同时**计算所有的 $7^x \pmod N$。而经典计算机需要计算每一个可能。
- **QFT 筛选答案**：Shor 算法利用 QFT 对计算出的并行结果进行处理。它会让错误的答案（非周期的波）相干抵消，让正确的答案（周期波）相长干涉。

### 周期查找和 QFT 在算法中的作用

周期查找是数学转化的桥梁，QFT是物理提取的工具。

周期查找利用数论知识，将分解大整数这个难题变为寻找函数 $f(x) = a^x \pmod N$ 的周期 $r$。然后利用量子叠加态，进行并行计算。

QFT 则是将周期查找里计算得到的答案进行**筛选**：排除非周期的错误答案，捕获正确的周期答案。

### 实践代码

目标：找到函数 $f(x) = 7^x \pmod{15}$ 的周期 $r$

预期：理论周期 $r=4$

**核心组件**：

1.  **Hadamard 门**：制造叠加态
2.  **Oracle**：实现 $7^x \pmod{15}$
3.  **IQFT **：提取周期

```python
import math
import numpy as np
from pyqpanda import *
from collections import Counter


def shor_period_finding_demo():
    print("=== Shor算法周期查找实战 (N=15, a=7) ===")

    machine = init_quantum_machine(QMachineType.CPU)

    # 资源分配
    # 测量寄存器: 用于存放指数 x，并读取周期信息
    # 理论上需要 t = 2*log2(N) 个，这里 N=15，取 t=3 个比特足以区分周期 4
    mq = machine.qAlloc_many(3)

    # 工作寄存器: 用于存放模幂运算结果 a^x mod N
    # 需要 log2(N) 个，即 4 个比特
    wq = machine.qAlloc_many(4)

    # 经典寄存器：用于保存测量结果
    c = machine.cAlloc_many(3)

    prog = QProg()

    # 初始化
    # 对测量寄存器应用 H 门，创造叠加态 |0> + ... + |7>
    prog.insert(single_gate_apply_to_all(H, mq))
    
    # 工作寄存器初始化为 |1> (即二进制 0001)，因为 a^0 = 1
    prog.insert(X(wq[0]))  

    # 模幂运算 Oracle (a^x mod N)
    # 需要实现 U|x>|1> = |x>|7^x mod 15>
    # 由于 x 是叠加态，这一步会并行计算所有的 7^x

    # a = 7, N = 15
    # 按位处理测量寄存器 mq 中的比特 (mq[0]是最低位 2^0, mq[1]是2^1...)
    # 逻辑：
    # 如果 mq[0]=1，工作寄存器乘以 7^1 = 7 mod 15
    # 如果 mq[1]=1，工作寄存器乘以 7^2 = 4 mod 15
    # 如果 mq[2]=1，工作寄存器乘以 7^4 = 1 mod 15 (恒等操作，不做任何事)

    # >> 子电路：受控乘 7
    # 7 * x mod 15 的效果是将 |1>->|7>, |7>->|4>, |4>->|13>, |13>->|1>
    # 使用等效门电路实现
    def swap_gate(q1, q2):
        return QCircuit().insert(SWAP(q1, q2))

    # 构建乘7的电路 (x7 mod 15)
    # 一个针对 N=15 的简化电路
    mult_7_circuit = QCircuit()
    mult_7_circuit.insert(SWAP(wq[0], wq[1]))
    mult_7_circuit.insert(SWAP(wq[1], wq[2]))
    mult_7_circuit.insert(SWAP(wq[2], wq[3]))
    mult_7_circuit.insert(single_gate_apply_to_all(X, wq))  # X门取反

    # 将乘7电路受控于 mq[0]
    prog.insert(mult_7_circuit.control(mq[0]))

    # >> 子电路：受控乘 4
    # 4 * x mod 15 (由 mq[1] 控制)
    # 乘 4 mod 15 的效果是简单的循环移位
    mult_4_circuit = QCircuit()
    mult_4_circuit.insert(SWAP(wq[0], wq[2]))
    mult_4_circuit.insert(SWAP(wq[1], wq[3]))

    # 将乘4电路受控于 mq[1]
    prog.insert(mult_4_circuit.control(mq[1]))

    # >> 子电路：受控乘 1
    # 乘 1 不改变状态，所以这里不需要插入任何门

    # 量子傅里叶逆变换
    # 对测量寄存器 mq 进行 IQFT，提取周期信息

    prog.insert(SWAP(mq[0], mq[2]))

    # H 和 受控相位门
    # H on mq[0]
    prog.insert(H(mq[0]))

    # CP on mq[0] controlled by mq[1] (angle -pi/2)
    prog.insert(U1(mq[0], -math.pi / 2).control(mq[1]))

    # CP on mq[0] controlled by mq[2] (angle -pi/4)
    prog.insert(U1(mq[0], -math.pi / 4).control(mq[2]))

    # H on mq[1]
    prog.insert(H(mq[1]))

    # CP on mq[1] controlled by mq[2] (angle -pi/2)
    prog.insert(U1(mq[1], -math.pi / 2).control(mq[2]))

    # H on mq[2]
    prog.insert(H(mq[2]))

    # 测量寄存器的结果
    prog.insert(measure_all(mq, c))

    shots = 1000
    result = machine.run_with_configuration(prog, c, shots)

    machine.finalize()
    return result
```

## Grover 搜索算法

### 定义

Grover 搜索算法 (Grover's Search Algorithm) 是由洛夫·格罗弗（Lov Grover）于 1996 年提出的量子算法。它旨在解决无序数据库搜索问题。

假设有这么一个问题：在一个包含 $N$ 个元素的无序数据库中，寻找满足特定条件函数 $f(x)=1$ 的唯一目标元素 $\omega$。

在经典算法中，由于数据无序，必须逐个检查，平均需要 $N/2$ 次查询，最坏情况需要 $N$ 次。时间复杂度为 $O(N)$。但在 Grover 算法中，利用**量子叠加和振幅放大**技术，只需约 $\frac{\pi}{4}\sqrt{N}$ 次查询即可找到目标。时间复杂度为 $O(\sqrt{N})$。

其核心机制就是**振幅放大**。不同于经典概率论，量子力学中的概率幅可以是负数。Grover 算法通过通过迭代调整概率幅，使目标态的概率幅不断增大（相干相长），非目标态的概率幅不断减小（相干相消），最终使系统以极高概率坍缩到目标态。

### Grover 算法的步骤

Grover 算法的执行过程可以概括为三个阶段的循环：

#### **1. 初始化**

将所有量子比特初始化为 $|0\rangle$，然后应用 Hadamard 门。

其目的是构建**均匀叠加态**。此时，数据库中所有 $N$ 个可能的索引值出现的概率完全相等，均为 $1/N$。

#### **2. 相位反转 Oracle $U_\omega$**

Oracle 是一个黑盒算子，它不需要知道目标 $\omega$ 是什么，但能识别它。

数学表达式为：
$$
U_\omega|x\rangle = (-1)^{f(x)}|x\rangle
$$

-  如果 $x$ 是非目标态，状态不变。
- 如果 $x$ 是目标态 $\omega$，其前面的系数（相位）翻转，变为负数。

其几何意义为**相位反转**。在几何表示中，这相当于将状态向量关于 "非目标集合" 平面进行反射。

#### **3. 扩散算子 $U_s$**

这是 Grover 算法中最巧妙的部分，也称为“关于平均值的反转”。

数学表达式为：
$$
U_s = 2|s\rangle\langle s| - I
$$
其中 $|s\rangle$ 是初始的均匀叠加态。

它的物理意义是将每个状态的概率幅围绕当前的 “平均概率幅” 进行翻转。由于 Oracle 刚刚把目标态的幅值变成了负数，通过这一步 “关于平均值翻转”，目标态的负幅值会被大幅度拉高成为正幅值，而非目标态的幅值会被压低。

### 示例：在 4 个抽屉中找东西 ($N=4$)

为了直观理解，我们使用 **2 个量子比特** ($2^2=4$) 来搜索目标 **$|11\rangle$** （即第4个元素）。

**初始状态**：系统处于 $|00\rangle$。

**步骤 1：均匀叠加**

应用 H 门。系统处于叠加态，4个状态的振幅都是 $0.5$（概率是 $0.5^2=0.25$）。

状态向量：$[0.5, 0.5, 0.5, 0.5]$，平均振幅：$0.5$。

**步骤 2：Oracle (标记 |11>)**

Oracle 识别出 $|11\rangle$ 是目标，将其相位翻转。此时状态向量变为：$[0.5, 0.5, 0.5, \mathbf{-0.5}]$，新的平均振幅变为 $(0.5+0.5+0.5-0.5)/4 = \mathbf{0.25}$。

**步骤 3：扩散算子（关于平均值反转）**

利用公式：$新振幅 = 2 \times 平均值 - 旧振幅$，此时平均值是 $0.25$。

 对于非目标 ($|00\rangle, |01\rangle, |10\rangle$)：$2 \times 0.25 - 0.5 = \mathbf{0}$

对于目标 ($|11\rangle$)：$2 \times 0.25 - (-0.5) = 0.5 + 0.5 = \mathbf{1}$

**结果**：

状态向量变为 $[0, 0, 0, 1]$，此时测量，我们将以 **100%** 的概率得到 $|11\rangle$。

*(注：对于 N=4，只需 1 次迭代即可完美找到；对于更大的 N，则概率接近 1 但不等于 1)*

### PyQPanda 实现

使用框架实现以上 $N=4$ 的示例，需要重点关注 oracle 和 diffusion 的构建方式。

```python
from pyqpanda import *
import numpy as np


def grover_search_demo():
    print("=== 开始 Grover 搜索算法 (N=4, 目标=11) ===")

    machine = init_quantum_machine(QMachineType.CPU)
    q = machine.qAlloc_many(2)  # 2个量子比特 -> 4个状态
    c = machine.cAlloc_many(2)  # 2个经典比特用于测量
    prog = QProg()

    # 阶段 1: 初始化 (均匀叠加)
    # Apply H gate to all qubits
    prog.insert(single_gate_apply_to_all(H, q))

    # 阶段 2: 构建 Oracle (标记 |11>)
    # 目标是翻转 |11> 的相位，即 |11> -> -|11>
    # 在量子电路中，Controlled-Z (CZ) 门的作用正是：当控制位和目标位都为1时，相位翻转。
    # 对于 2比特系统，CZ(q0, q1) 等价于 Oracle。
    # PyQPanda中 Z(q1).control(q0) 即为 CZ。
    prog.insert(Z(q[1]).control(q[0]))

    # 阶段 3: 扩散算子 (Diffusion / Inversion about mean)
    # 标准电路公式：H -> X -> H(n-1)Z -> X -> H
    # 对于2比特，这可以简化为：
    
    # 1. H 门变换
    prog.insert(single_gate_apply_to_all(H, q))

    # 2. X 门变换 (翻转0和1，为了配合后面的控制门)
    prog.insert(single_gate_apply_to_all(X, q))

    # 3. 多控Z门 (H(n-1)Z)
    # 对于2比特，这里只需要一个 CZ 门
    prog.insert(Z(q[1]).control(q[0]))

    # 4. X 门恢复
    prog.insert(single_gate_apply_to_all(X, q))

    # 5. H 门恢复
    prog.insert(single_gate_apply_to_all(H, q))

    prog.insert(measure_all(q, c))

    # 运行验证 
    # 理论上，N=4 时运行一次迭代后概率应为 100%
    result = machine.run_with_configuration(prog, c, 1000)
    print("搜索结果分布:", result)

    machine.finalize()


if __name__ == "__main__":
    grover_search_demo()
```

### 平法级加速的原因

因为 Grover 算法本质上是一个**旋转**过程，而不是筛选过程。

可以把量子系统的状态看作一个多维空间中的**矢量**。

**初始状态**： $|s\rangle$（均匀叠加态）主要指向非目标的方向，只有极小一点分量指向目标方向。

**目标**：把这个矢量旋转，让它尽可能指向目标状态 $|\omega\rangle$

**每一次迭代（Oracle + 扩散算子）**的作用就是要让状态矢量向目标方向旋转一个固定的角度 $\theta$。数学上可以证明，这个角度 $\theta \approx \frac{2}{\sqrt{N}}$。

从起点（几乎垂直于目标）转到终点（平行于目标），总共需要转 $90^\circ$（即 $\frac{\pi}{2}$ 弧度）。

而**要旋转的次数（步数）**为：
$$
k \approx \frac{\text{总路程}}{\text{单步跨度}} = \frac{\pi/2}{2/\sqrt{N}} \approx \frac{\pi}{4}\sqrt{N}
$$
**结论**：因为每次迭代旋转的角度与 $\frac{1}{\sqrt{N}}$ 成正比，而不是 $\frac{1}{N}$，所以总步数就是 $\sqrt{N}$ 级别。这就是平方级加速的数学几何本质。

### Oracle 的作用

Oracle 是整个算法的判决器，但它是一个瞎子判决器，不知道目标是什么。

它的功能是**相位反转**。它不改变任何状态的概率（振幅的大小），只改变目标状态的**符号**（相位）。 如果目标是 $x$，Oracle 会把 $|x\rangle$ 变成 $-|x\rangle$，其他所有非目标状态保持不变。

而由于量子测量的概率是振幅的平方（$|Amplitude|^2$），如果只运行 Oracle 就去测量，目标出现的概率和原来一模一样。**单纯的标记在测量时是看不见的。**

例如：$|0.5|^2 = 0.25$， $|-0.5|^2 = 0.25$。

因此还需要扩散算子**将振幅放大**。

### 扩散算子的作用

扩散算子是 Grover 算法的放大镜，它把 Oracle 做的隐形记号变成了“显形的高度”。

它主要的功能是**关于平均值的反转**。首先计算当前所有状态振幅的**平均值**，然后将每个状态的振幅翻转到平均值的另一侧。公式：$新值 = 2 \times 平均值 - 旧值$

Oracle 刚做完时，大部分数据是正的（非目标），只有一个是负的（目标）。此时，整体的**平均值**会被这个负数拉低，变得非常小（但在正数那边）。

**此时扩散算子介入**：

- 非目标（正值）：它们比平均值高一点点。翻转后，它们变到了平均值下面一点点，即振幅变小。
- 目标（负值）：它离平均值非常远（在下面）。翻转后，它会跳到平均值上方非常远的地方，即振幅**剧烈变大**。

总之，**Oracle** 负责将目标与众不同（通过变负号），**扩散算子** 利用这个负号，通过数学技巧吸走其他人的概率，全部转移给目标，从而实现大概率的测量结果。

### PyQPanda 实现 Grover 搜索 |11>

任务介绍：通过PyQPanda实现Grover搜索算法的量子电路，搜索特定目标状态，并验证其正确性。

任务目标：能够用PyQPanda实现Grover算法并运行。输出：完成一个可运行的量子电路代码。

任务要求：可运行并正确搜索11 状态。运行代码，记录测量结果（应高概率得到11）。

```python
from pyqpanda import *


def grover_search_target_11():
    """
    使用 PyQPanda 实现 2比特 Grover 搜索算法
    目标状态: |11>
    """

    # 初始化量子虚拟机 (CPU模式)
    machine = init_quantum_machine(QMachineType.CPU)

    # 申请 2 个量子比特 (q[0], q[1])
    q = machine.qAlloc_many(2)
    # 申请 2 个经典比特用于保存测量结果
    c = machine.cAlloc_many(2)

    prog = QProg()

    # 步骤一：均匀叠加
    # 对所有量子比特应用 Hadamard 门
    # 状态从 |00> 变为 (|00> + |01> + |10> + |11>) / 2
    prog.insert(single_gate_apply_to_all(H, q))

    # 步骤二：Oracle (相位反转)
    # 目标是 |11>
    # 需要一个算子，仅当 q[0]=1 且 q[1]=1 时，翻转相位。
    # 也就是 Z(target).control(control)。
    # 这里的 CZ 门正好满足：只对 |11> 变号。
    prog.insert(Z(q[1]).control(q[0]))

    # 步骤三：扩散算子
    # 作用：围绕平均值翻转振幅，放大目标态概率。
    # 标准电路公式：H -> X -> (多控Z) -> X -> H

    # 1. H 门
    prog.insert(single_gate_apply_to_all(H, q))

    # 2. X 门
    prog.insert(single_gate_apply_to_all(X, q))

    # 3. 多控 Z 门 (对于2比特，就是 CZ)
    # 这步配合前后的 X 门，实现了对 |00> 的相位翻转，从而完成扩散逻辑
    prog.insert(Z(q[1]).control(q[0]))

    # 4. X 门 (恢复)
    prog.insert(single_gate_apply_to_all(X, q))

    # 5. H 门 (恢复)
    prog.insert(single_gate_apply_to_all(H, q))

    # 测量
    # 将量子态坍缩为经典比特
    prog.insert(measure_all(q, c))

    print("开始运行 Grover 搜索 (目标: 11)...")
    # 运行 1000 次 (shots=1000)
    result = machine.run_with_configuration(prog, c, 1000)

    print("\n=== 测量结果统计 ===")
    print(result)

    # 简单的自动验证逻辑
    target_count = result.get('11', 0)
    if target_count > 900:
        print(f"\n[成功] 目标状态 '11' 出现的频率极高 ({target_count}/1000)，算法运行正确！")
    else:
        print("\n[失败] 目标状态出现频率不足，请检查电路。")

    machine.finalize()


if __name__ == "__main__":
    grover_search_target_11()
```

```bash
{'11': 5000}

[成功] 目标状态 '11' 出现的频率极高 (5000/5000)，算法运行正确！
```

**代码关键点**

**Oracle 的选择**：

代码中使用了 `Z(q[1]).control(q[0])`。这是因为 $Z$ 门的作用矩阵是 $\begin{bmatrix} 1 & 0 \\ 0 & -1 \end{bmatrix}$，只有当比特为 1 时才变负号。加上控制位后，只有当两个比特都是 1 时，整体相位才会变负。这精确地标记了 $|11\rangle$。

**扩散算子结构**：`H -> X -> CZ -> X -> H` 是实现 $2|s\rangle\langle s| - I$ 的标准通用电路模板。

## 量子机器学习

### 定义

顾名思义，QML 就是**量子计算 + ML**。它利用量子计算机的特性（叠加、纠缠、干涉）来改进或加速经典的机器学习算法，或者利用机器学习算法来分析量子系统。

因为 ML 的模型例如神经网络、SVM，本质上就是在进行海量的矩阵和向量运算。而巧的是量子计算做的事就是**指数级加速**矩阵（逻辑门）和向量（量子态）运行。所以未来使用量子计算机加速训练大模型是非常有前景的。

###  QML 的四种形态

根据数据来源（经典/量子）和处理设备（经典/量子），QML 可以分为四类。大家目前最关注的是 **CQ** 类型：

| 数据类型          | 算法/设备         | 描述                 | 例子                                                  |
| :---------------- | :---------------- | :------------------- | :---------------------------------------------------- |
| **C** (Classical) | **C** (Classical) | **经典机器学习**     | 在CPU上跑神经网络识别猫狗                             |
| **Q** (Quantum)   | **C** (Classical) | **量子态层析**       | 用经典电脑分析量子实验数据                            |
| **C** (Classical) | **Q** (Quantum)   | **量子增强机器学习** | 👉️用量子计算机加速处理经典数据（如金融预测、图像分类） |
| **Q** (Quantum)   | **Q** (Quantum)   | **全量子机器学习**   | 用量子电脑处理量子数据（如直接处理光子信号）          |

### 为什么量子计算能加速 ML 的训练

#### 1. 量子特征映射
在 SVM（支持向量机）中，为了把纠缠在一起的数据分开，我们需要把数据映射到更高维的空间。

在经典计算机中计算高维内积（Kernel Trick）计算量很大。但在**量子计算机**里，通过量子编码，可以将经典数据 $x$ 映射为**量子态** $|\phi(x)\rangle$。利用希尔伯特空间的无限维特性，量子计算机可以轻松在高维空间中找到分类超平面。

#### 2. 变分量子电路 (VQC) —— 量子神经网络
这是目前 NISQ（含噪声中等规模量子）时代的主流架构。它长得非常像经典神经网络，但是把 **权重** 换成了 **量子门的旋转角度 ($\theta$)**

*   **输入层**：把经典数据编码成量子态。
*   **隐藏层**：由带参数的量子门（如 $R_x(\theta), R_y(\theta)$）和纠缠门（CNOT）组成。
*   **输出层**：测量量子比特，得到结果。
*   **训练**：测量结果与真实标签对比 -> 算出损失 -> 经典优化器（如梯度下降）调整量子门的角度 $\theta$ -> 循环。

### VQE 

VQE 是 QML 思想的一个典型应用，也是目前量子计算机最有希望最早实现商业价值的算法，尤其是在用于化学分子模拟领域。

问题背景：你想知道一个药物分子（比如咖啡因）在自然界中最稳定的状态不仅是怎样的（基态能量是多少）。经典电脑算不动，因为电子之间的相互作用太复杂。

**VQE 的工作原理**：

- 量子计算机：

  量子计算机生成一个用于猜测结果的波函数 $|\psi(\theta)\rangle$（由一组参数 $\theta$ 控制），并测量这个状态的能量。

- 经典计算机：

  经典计算机负责读取能量值。如果能量还不够低，经典优化器（如梯度下降）会告诉量子计算机：“把参数 $\theta$ 稍微调大一点试试”。

在二者不断地配合和循环下，直到找到能量地最低点（全局最小值），也称本征值。

### PyQPanda 实现简单哈密顿量的 VQE

#### **目标**

寻找单量子比特系统在哈密顿量 $H = \sigma_z$ 下的基态（能量最低态）。

**理论预期**：

*   哈密顿量 $H = Z$
*   本征态为 $|0\rangle$ (能量 +1) 和 $|1\rangle$ (能量 -1)
*   我们的目标是让量子计算机自动找到参数，制备出 $|1\rangle$ 态，使得能量达到最低值 **-1**

**电路设计**：

*   初始状态：$|0\rangle$
*   变分门：$R_y(\theta)$（通过旋转 $\theta$ 角度来改变状态）

#### **PyQPanda VQE 实战代码**

```python
import numpy as np
from pyqpanda import *
from scipy.optimize import minimize
import matplotlib.pyplot as plt

# 环境准备
# 初始化量子虚拟机
machine = init_quantum_machine(QMachineType.CPU)

# 全局变量记录优化过程
energy_history = []

q = machine.qAlloc_many(1)
c = machine.cAlloc_many(1)

def get_variational_circuit(q, theta):
    """
    构建变分量子电路 (Ansatz)
    参数:
        q: 量子比特列表
        theta: 变分参数 (旋转角度)
    返回:
        QProg: 量子程序
    """
    prog = QProg()

    # 变分部分：在 q[0] 上施加 Ry 旋转门
    # Ry 门可以将 |0> 旋转向 |1>
    prog.insert(RY(q[0], theta))

    return prog


def cost_function(params):
    """
    代价函数 (计算能量期望值)
    VQE 的目标是最小化这个函数的返回值
    """
    theta = params[0]

    # 获取电路
    prog = get_variational_circuit(q, theta)

    # 计算期望值 <Z>
    # 计算 P(0) 和 P(1)
    # 期望值 <Z> = 1 * P(0) + (-1) * P(1)

    # 使用 prob_run_dict 直接获取理论概率
    # 这种方式对于优化器更平滑，适合模拟器演示
    result = machine.prob_run_dict(prog, q, -1)

    p0 = result.get('0', 0)  # 测量为0的概率
    p1 = result.get('1', 0)

    # 计算能量
    # H = Z matrix: diag(1, -1)
    energy = 1.0 * p0 + (-1.0) * p1

    # 记录历史用于画图
    energy_history.append(energy)

    return energy


def run_vqe():
    print("==== 开始运行简单的 VQE 演示 ====")
    print("目标哈密顿量: H = Z")
    print("理论最低能量 (基态): -1.0 (对应状态 |1>)")

    # 初始参数猜测
    # 故意猜一个离目标很远的值，比如 0.0 (对应状态 |0>, 能量 +1)
    init_params = np.array([0.0])
    print(f"初始参数 theta: {init_params[0]}")
    print(f"初始能量: {cost_function(init_params)} (记录中...)")

    # 清空因为打印初始能量而产生的记录
    energy_history.clear()

    # 使用经典优化器进行训练
    # 使用 COBYLA 方法，因为它不需要计算梯度，适合含噪环境
    print("\n开始经典优化 (Scipy COBYLA)...")
    res = minimize(cost_function,
                   init_params,
                   method='COBYLA',
                   tol=1e-4,
                   options={'maxiter': 100})

    # 输出结果
    print("\n==== 优化完成 ====")
    print(f"是否收敛: {res.success}")
    print(f"迭代次数: {res.nfev}")
    print(f"最优参数 theta: {res.x[0]:.4f} (理论上应接近 PI = 3.14159)")
    print(f"最优能量: {res.fun:.6f}")

    # 验证结果
    # 如果 theta 接近 PI (3.14)，说明 Ry(pi)|0> = |1>，完全正确
    error = abs(res.x[0] - np.pi)
    if error < 0.1:
        print(">> 成功！电路已找到基态 |1>。")
    else:
        print(">> 结果偏差较大，可能陷入局部最优或迭代不足。")

    # 简单的文本绘图
    print("\n[能量下降曲线]")
    for i, e in enumerate(energy_history):
        if i % 2 == 0:
            bar_len = int((e + 1) * 10)  # 简单的可视化缩放
            bar = "*" * bar_len
            print(f"Iter {i:02d}: Energy = {e:+.4f} {bar}")

    machine.finalize()


if __name__ == "__main__":
    run_vqe()
```
